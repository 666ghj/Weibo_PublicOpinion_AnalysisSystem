# 小红书热搜爬取使用指南

## 🎯 功能概述

MediaCrawler 现已支持小红书热搜爬取功能，可以尝试获取小红书热门内容和关键词，并可选择性地爬取热搜相关的笔记内容。

⚠️ **重要说明**: 目前小红书的热搜API接口访问受限，可能需要特殊的登录状态或API已发生变更。本功能主要作为技术实现参考。

## ✨ 功能特性

- **多种获取方式**：尝试多个API接口，提高成功率
- **热搜数据保存**：支持多种存储格式（CSV、JSON、数据库、SQLite）
- **热搜笔记爬取**：可选择性爬取热搜关键词相关的笔记
- **数据结构完整**：包含热搜关键词、排名、描述、爬取时间等信息
- **配置灵活**：支持自定义热搜数量、笔记数量等参数
- **真实数据优先**：仅爬取真实热搜数据，不使用默认关键词

## 🚀 快速开始

### 1. 基础配置

在 `config/base_config.py` 中设置以下配置：

```python
# 设置平台为小红书
PLATFORM = "xhs"

# 设置爬取类型为热搜
CRAWLER_TYPE = "trending"

# 热搜相关配置
ENABLE_TRENDING_KEYWORDS_CRAWL = True  # 启用热搜关键词爬取
TRENDING_KEYWORDS_COUNT = 10           # 热搜关键词数量限制
ENABLE_TRENDING_POSTS_CRAWL = True     # 是否爬取热搜相关笔记
TRENDING_KEYWORD_MAX_NOTES_COUNT = 20  # 每个热搜关键词爬取的笔记数量
```

### 2. 运行爬虫

```bash
python main.py
```

### 3. 数据存储

热搜数据将根据 `SAVE_DATA_OPTION` 配置保存到相应的存储中：

- **JSON格式**: `data/xhs/json/xhs_notes_*.json`
- **CSV格式**: `data/xhs/csv/xhs_notes_*.csv`
- **数据库**: `xhs_note` 表
- **SQLite**: `data/xhs/xhs_notes.db`

## 🔧 高级配置

### 1. 仅获取热搜关键词（不爬取相关笔记）

```python
ENABLE_TRENDING_POSTS_CRAWL = False
```

### 2. 自定义热搜数量

```python
TRENDING_KEYWORDS_COUNT = 50  # 获取前50个热搜
```

### 3. 批量爬取热搜笔记

```python
ENABLE_TRENDING_POSTS_CRAWL = True
TRENDING_KEYWORD_MAX_NOTES_COUNT = 100  # 每个热搜爬取100个笔记
```

## 📝 注意事项

1. **API限制**：小红书对API访问有一定限制，建议合理控制爬取频率
2. **登录状态**：某些热搜API可能需要登录状态，建议先完成登录
3. **数据去重**：相同时间的热搜关键词会自动去重
4. **错误处理**：API失败时会自动尝试备用接口
5. **存储优化**：支持增量更新，避免重复存储

## 🔍 故障排除

### 常见问题

1. **热搜数据为空**
   - 检查网络连接
   - 确认登录状态是否有效
   - 查看日志中的错误信息
   - **当前状态**: 小红书热搜API可能需要特殊权限或已变更

2. **JSON解析错误**
   - 小红书API可能返回非JSON格式数据
   - 可能需要特殊的请求头或认证
   - 建议检查API响应的原始内容

3. **API接口失败**
   - 小红书可能已更新API接口
   - 需要分析最新的网页结构和API调用
   - 考虑使用页面抓取方式作为备选方案

4. **存储失败**
   - 检查数据库连接
   - 确认表结构是否正确创建

### 调试方法

1. 启用详细日志：
   ```python
   ENABLE_VERBOSE_LOGGING = True
   ```

2. 使用无头模式调试：
   ```python
   HEADLESS = False
   ```

3. 检查API响应：
   ```python
   # 在代码中添加调试信息
   utils.logger.info(f"API response: {trending_res}")
   ```

## 🔄 API接口说明

小红书热搜爬取使用了多个备用API接口：

1. **主要接口**: `/api/sns/web/v1/search/trending`
2. **备用接口1**: `/api/sns/web/v1/search/suggest`
3. **备用接口2**: `/api/sns/web/v1/homefeed` (发现页面热门内容)

如果主要接口失败，系统会自动尝试备用接口，确保数据获取的稳定性。

## 📈 性能优化

1. **并发控制**: 通过 `MAX_CONCURRENCY_NUM` 控制并发数量
2. **延时设置**: 通过 `CRAWLER_MAX_SLEEP_SEC` 控制爬取间隔
3. **代理支持**: 支持IP代理池，提高爬取成功率
