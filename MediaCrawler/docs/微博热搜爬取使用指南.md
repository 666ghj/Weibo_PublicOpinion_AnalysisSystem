# 微博热搜爬取使用指南

## 🎯 功能概述

MediaCrawler 现已支持微博热搜爬取功能，可以自动获取微博实时热搜榜单，并可选择性地爬取热搜相关的帖子内容。

## ✨ 功能特性

- **实时热搜获取**：自动获取微博实时热搜榜单
- **热搜数据保存**：支持多种存储格式（CSV、JSON、数据库、SQLite）
- **热搜帖子爬取**：可选择性爬取热搜关键词相关的帖子
- **数据结构完整**：包含热搜关键词、排名、描述、爬取时间等信息
- **配置灵活**：支持自定义热搜数量、帖子数量等参数

## 🚀 快速开始

### 1. 基础配置

在 `config/base_config.py` 中设置以下配置：

```python
# 设置平台为微博
PLATFORM = "wb"

# 设置爬取类型为热搜
CRAWLER_TYPE = "trending"

# 热搜相关配置
ENABLE_TRENDING_KEYWORDS_CRAWL = True  # 启用热搜关键词爬取
TRENDING_KEYWORDS_COUNT = 10           # 热搜关键词数量限制
ENABLE_TRENDING_POSTS_CRAWL = True     # 是否爬取热搜相关帖子
TRENDING_KEYWORD_MAX_NOTES_COUNT = 20  # 每个热搜关键词爬取的帖子数量
```

### 2. 运行爬虫

```bash
python main.py
```

### 3. 数据存储

热搜数据将根据 `SAVE_DATA_OPTION` 配置保存到相应的存储中：

- **CSV**: `data/weibo/trending_*.csv`
- **JSON**: `data/weibo/json/trending_*.json`
- **数据库**: `weibo_trending` 表
- **SQLite**: `weibo_trending` 表

## 📊 数据结构

### 热搜数据表结构

```sql
CREATE TABLE `weibo_trending`
(
    `id`               int         NOT NULL AUTO_INCREMENT COMMENT '自增ID',
    `keyword`          varchar(255) NOT NULL COMMENT '热搜关键词',
    `desc`             varchar(255) DEFAULT NULL COMMENT '热搜描述（包含标记）',
    `rank`             int         NOT NULL COMMENT '热搜排名',
    `platform`         varchar(32) NOT NULL DEFAULT 'weibo' COMMENT '平台名称',
    `crawl_time`       bigint      NOT NULL COMMENT '爬取时间戳',
    `crawl_date_time`  varchar(32) NOT NULL COMMENT '爬取日期时间',
    `last_modify_ts`   bigint      NOT NULL COMMENT '最后修改时间戳',
    PRIMARY KEY (`id`),
    KEY                `idx_weibo_trending_keyword` (`keyword`),
    KEY                `idx_weibo_trending_rank` (`rank`),
    KEY                `idx_weibo_trending_crawl_time` (`crawl_time`)
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4 COLLATE=utf8mb4_0900_ai_ci COMMENT='微博热搜关键词';
```

## ⚙️ 配置参数详解

### 基础配置

| 参数 | 类型 | 默认值 | 说明 |
|------|------|--------|------|
| `PLATFORM` | str | "wb" | 平台设置，微博为"wb" |
| `CRAWLER_TYPE` | str | "trending" | 爬取类型，热搜为"trending" |

### 热搜专用配置

| 参数 | 类型 | 默认值 | 说明 |
|------|------|--------|------|
| `ENABLE_TRENDING_KEYWORDS_CRAWL` | bool | True | 是否启用热搜关键词爬取 |
| `TRENDING_KEYWORDS_COUNT` | int | 10 | 热搜关键词数量限制 |
| `ENABLE_TRENDING_POSTS_CRAWL` | bool | True | 是否爬取热搜相关帖子 |
| `TRENDING_KEYWORD_MAX_NOTES_COUNT` | int | 20 | 每个热搜关键词爬取的帖子数量 |

### 通用配置

| 参数 | 类型 | 默认值 | 说明 |
|------|------|--------|------|
| `SAVE_DATA_OPTION` | str | "db" | 数据保存方式：csv/json/db/sqlite |
| `HEADLESS` | bool | False | 是否无头模式运行 |
| `ENABLE_GET_COMMENTS` | bool | True | 是否爬取评论 |

## 🔧 高级用法

### 1. 仅获取热搜榜单（不爬取帖子）

```python
ENABLE_TRENDING_POSTS_CRAWL = False
```

### 2. 自定义热搜数量

```python
TRENDING_KEYWORDS_COUNT = 50  # 获取前50个热搜
```

### 3. 批量爬取热搜帖子

```python
ENABLE_TRENDING_POSTS_CRAWL = True
TRENDING_KEYWORD_MAX_NOTES_COUNT = 100  # 每个热搜爬取100个帖子
```

## 📝 注意事项

1. **频率控制**：热搜爬取会自动添加延时，避免请求过快
2. **数据去重**：相同时间的热搜关键词会自动去重
3. **错误处理**：API失败时会自动尝试备用接口
4. **存储优化**：支持增量更新，避免重复存储

## 🔍 故障排除

### 常见问题

1. **热搜数据为空**
   - 检查网络连接
   - 确认API接口可访问
   - 查看日志中的错误信息

2. **解析失败**
   - 微博API结构可能发生变化
   - 检查解析逻辑是否需要更新

3. **存储失败**
   - 检查数据库连接
   - 确认表结构是否正确创建

### 调试方法

1. 启用详细日志：
   ```python
   ENABLE_VERBOSE_LOGGING = True
   ```

2. 运行测试脚本查看详细输出：
   ```bash
   python test_trending_standalone.py
   ```
