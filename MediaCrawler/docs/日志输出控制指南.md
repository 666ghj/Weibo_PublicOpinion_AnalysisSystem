# MediaCrawler 日志输出控制指南

## 概述

MediaCrawler 提供了灵活的日志输出控制选项，帮助用户根据需要调整日志的详细程度，减少不必要的信息干扰。

## 日志级别配置

### 简化日志模式（推荐）

默认情况下，MediaCrawler 使用简化日志模式，只显示关键信息：

```python
# config/base_config.py
ENABLE_VERBOSE_LOGGING = False  # 默认值
```

**简化模式特点：**
- ✅ 只显示爬取进度和重要状态信息
- ✅ 隐藏HTTP请求详细日志
- ✅ 隐藏asyncio和playwright的调试信息
- ✅ 减少终端输出干扰

**示例输出：**
```
2025-08-14 12:00:00 MediaCrawler INFO - Begin search xiaohongshu trending keywords
2025-08-14 12:00:01 MediaCrawler INFO - Found 20 trending keywords: ['美妆', '护肤', '穿搭'...]
2025-08-14 12:00:02 MediaCrawler INFO - (1/20) Searching trending keyword: 美妆
2025-08-14 12:00:05 MediaCrawler INFO - keyword: 美妆, crawled: 20/50
2025-08-14 12:00:06 MediaCrawler INFO - (2/20) Searching trending keyword: 护肤
```

### 详细日志模式

如果需要调试或查看详细的网络请求信息，可以启用详细日志模式：

```python
# config/base_config.py
ENABLE_VERBOSE_LOGGING = True
```

**详细模式特点：**
- 📝 显示所有HTTP请求和响应
- 📝 显示asyncio任务状态
- 📝 显示playwright浏览器操作
- 📝 适合开发调试使用

## 常见日志信息说明

### 正常运行日志

| 日志信息 | 说明 |
|---------|------|
| `Begin search xiaohongshu trending keywords` | 开始获取小红书热搜关键词 |
| `Found X trending keywords` | 成功获取到X个热搜关键词 |
| `(X/Y) Searching trending keyword: 关键词` | 正在搜索第X个关键词（共Y个） |
| `keyword: 关键词, crawled: X/Y` | 关键词爬取进度：已爬取X个，目标Y个 |
| `Begin batch get note comments` | 开始批量获取笔记评论 |
| `Crawler finished` | 爬取任务完成 |

### 错误和警告日志

| 日志信息 | 说明 | 处理方式 |
|---------|------|----------|
| `Error getting trending keywords` | 获取热搜关键词失败 | 检查网络连接和登录状态 |
| `Error searching keyword` | 搜索关键词失败 | 可能是网络问题或API限制 |
| `No trending keywords found` | 未找到热搜关键词 | 检查API响应格式 |
| `RetryError` | 重试失败 | 通常是网络或服务器问题 |

### 程序结束时的异常信息

程序结束时可能出现的异常信息（可以忽略）：

```
asyncio ERROR - Task was destroyed but it is pending!
RuntimeError: Event loop is closed
TargetClosedError: Target page, context or browser has been closed
```

这些错误是程序正常结束时的清理过程产生的，不影响数据爬取结果。

## 自定义日志配置

### 修改日志格式

如果需要自定义日志格式，可以修改 `tools/utils.py` 中的 `init_loging_config` 函数：

```python
def init_loging_config():
    level = logging.INFO
    logging.basicConfig(
        level=level,
        format="%(asctime)s %(levelname)s - %(message)s",  # 简化格式
        datefmt='%Y-%m-%d %H:%M:%S'
    )
    # ... 其他配置
```

### 调整特定模块的日志级别

```python
# 只显示ERROR级别的httpx日志
httpx_logger = logging.getLogger("httpx")
httpx_logger.setLevel(logging.ERROR)

# 完全禁用某个模块的日志
some_logger = logging.getLogger("module_name")
some_logger.disabled = True
```

## 日志文件输出

如果需要将日志保存到文件，可以修改日志配置：

```python
import logging
from logging.handlers import RotatingFileHandler

def init_loging_config():
    # 创建文件处理器
    file_handler = RotatingFileHandler(
        'mediacrawler.log', 
        maxBytes=10*1024*1024,  # 10MB
        backupCount=5
    )
    file_handler.setFormatter(logging.Formatter(
        "%(asctime)s %(name)s %(levelname)s - %(message)s"
    ))
    
    # 配置根日志器
    logging.basicConfig(
        level=logging.INFO,
        handlers=[
            logging.StreamHandler(),  # 控制台输出
            file_handler             # 文件输出
        ]
    )
```

## 总结

- **默认使用简化日志模式**，减少信息干扰
- **程序结束时的异常信息可以忽略**，不影响爬取结果
- **如需调试，启用详细日志模式**
- **关注关键的进度和状态信息**，了解爬取进展

通过合理的日志配置，您可以获得清晰、有用的程序运行信息，同时避免被大量冗余日志干扰。
